代码风格参照这个:
后端（FastAPI）
分层架构，六层职责清晰：
routers/ — 路由层，只负责参数提取和调用 crud，不写 SQL
schemas/ — Pydantic v2 模型，统一用 ConfigDict(from_attributes=True, populate_by_name=True) + Field(alias=...) 做字段映射（Python snake_case ↔ 前端 camelCase）
models/ — SQLAlchemy 2.0 声明式模型，Mapped + mapped_column 风格，每个模型有 __table_args__ 索引，comment 注释完整
crud/ — 纯异步数据库操作层，只接受 AsyncSession，返回 Python 对象
utils/ — 工具模块，包含 auth.py（token 鉴权）、security.py（bcrypt 密码）、response.py（统一响应）、exception.py / exception_handlers.py（全局异常注册）
config/ — 数据库与 Redis 连接配置
cache/ — Redis 缓存封装，序列化/反序列化
风格特征：
异步贯穿始终 — async / await、AsyncSession、aiomysql、redis.asyncio，每个 crud 函数都是 async def
统一响应格式 — 所有接口返回 { "code": 200, "message": "...", "data": ... }，通过 success_response() 工具函数构建
鉴权方式 — UUID token 存 user_token 表，Authorization 请求头传入，用 get_current_user 依赖注入
异常处理 — 全局注册了 HTTPException、IntegrityError、SQLAlchemyError、Exception 四个级别的处理器，开发模式下返回详细错误追踪
编码风格 — 中文字段备注（SQL 注释、API 描述），部分文件有大量尾部空行，代码间距和命名一致性还有一些磨合空间（remover_all_favorites typo、update_user 中 update_user=await get_user_by_username 变量名覆盖等），整体风格接近黑马/传智教学项目的写法，功能性完整，工整度在演进中
前端（Vue 3 + Vite）
UI 框架：Vant 4（移动端组件库），面向手机 H5
状态管理：Pinia + pinia-plugin-persistedstate（持久化）
路由：vue-router
HTTP 请求：axios
构建工具：Vite 7
其他：marked + dompurify（Markdown 渲染/安全过滤）、vue-i18n（国际化）
项目风格一句话
一个典型的黑马教学风格全栈项目：FastAPI 异步分层后端 + Vue 3 移动端 H5 前端，功能覆盖新闻列表/分类/详情、用户注册登录、收藏、浏览历史，代码量适中、结构规整，注释偏中文教学导向。
